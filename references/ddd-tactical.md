# DDD 战术模式速查

## 核心概念

### 聚合 (Aggregate)
- 一组相关对象的簇，通过**聚合根**访问
- 聚合是唯一一致性边界，所有修改必须通过聚合根
- 不变约束在聚合内维护，跨聚合一致性通过最终一致性实现

### 实体 (Entity)
- 有唯一标识和生命周期的对象
- 通过 ID 识别，不是通过属性值

### 值对象 (Value Object)
- 无标识，仅通过属性值区分
- 不可变（immutable）
- 值相等性：属性相同即相等

### 领域服务 (Domain Service)
- 无状态操作
- 不属于任何聚合的逻辑（跨聚合协调、复杂计算）

### 领域事件 (Domain Event)
- 领域中发生的重要事情
- 用于跨聚合/上下文通信
- 命名：过去式（OrderPlaced, PaymentReceived）

### 仓库 (Repository)
- 每个聚合对应一个仓库
- 提供面向应用的集合式接口

### 防腐层 (Anti-Corruption Layer)
- 隔离外部系统/遗留系统的模型
- 保护内部领域模型不被外部概念污染

## 设计原则

1. **保持聚合小** — 不超过 3 个子对象
2. **值对象优先** — 能是 VO 就不用 Entity（减少复杂度）
3. **聚合边界 = 事务边界** — 一个聚合在一个一致性边界内
4. **引用 ID 不引用实例** — 跨聚合只引用 ID，不持有对象引用
5. **领域逻辑在领域层** — 拒绝贫血模型，行为属于领域对象
